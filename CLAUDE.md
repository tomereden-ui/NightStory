# NightStory — Claude Context

Bedtime story app for families. Parents create AI-generated audio stories personalised to their child. Built with Next.js 14 App Router, Supabase, Gemini, and ElevenLabs.

## Stack

- **Framework**: Next.js 14 App Router (`"use client"` where needed)
- **Styling**: Tailwind CSS + CSS custom properties for the dynamic type scale
- **Database / Storage**: Supabase (Postgres + Storage buckets)
- **AI — image generation**: Gemini `gemini-2.5-flash-image` (server-side, via `/api/admin/seed-create-images`)
- **AI — story generation**: Gemini (see `/api/generate-story`, `/api/five-question-story`)
- **TTS**: ElevenLabs (`/api/synthesize-speech`, `/api/el-voices`)
- **Branch in use**: `claude/nightstory-nextjs-setup-0lhBG`
- **Push command** (PAT required in this env):


## Typography System

All text sizes use CSS custom properties set by `FontSizeContext`. Never hardcode px font sizes.

**Tailwind classes**: `text-fs-display`, `text-fs-title`, `text-fs-subtitle`, `text-fs-heading`, `text-fs-body`, `text-fs-label`, `text-fs-caption`, `text-fs-micro`

**Or inline**: `fontSize: "var(--fs-body)"` etc.

**Scale at Medium (1.0×)**:
| Token | px | Use |
|---|---|---|
| display | 32 | Hero greeting, single-line massive title |
| title | 24 | Page/story titles |
| subtitle | 20 | Section/modal titles, prominent labels |
| heading | 17 | Sub-section headers, form labels |
| body | 14 | Body text, descriptions, chat messages |
| label | 12 | UI labels, tabs, small buttons |
| caption | 10 | Timestamps, metadata |
| micro | 9 | Badge pills, avatar name chips |

Multipliers: small 0.85×, medium 1.0×, large 1.2×. **Minimum visible size is 14px (body) — nothing important should use caption/micro/label for real content.**

## Image Generation (Creation Flow)

Option card images for the 5-question wizard are generated by Gemini and cached in Supabase bucket `story-options`.

**Storage key format**: `v5-{type}-{id}.jpg` (e.g. `v5-world-deep-ocean.jpg`)
- The `v5` prefix is a version key — bump it (→ v6, v7…) whenever prompts change to force regeneration
- All prompts live in `src/config/createFlowImages.ts`
- The seeder API is `POST /api/admin/seed-create-images` (strips version prefix before uploading)

**Browser-side flow in `FiveQuestionFlow.tsx`**:
1. On mount, read `localStorage` key `ns-option-images-v5` → show images instantly if cached
2. Compare cached keys against all expected specs; POST missing ones to the seeder in batches of 8
3. Show pulsing "Painting images…" badge (`imagesGenerating` state) while batch is running
4. Save each result back to `localStorage` immediately

When bumping the version, update **both** `optionStorageKey()` in `createFlowImages.ts` **and** `LS_KEY` / `optionStorageKey` references in `FiveQuestionFlow.tsx`.

## Avatar Visual Language

Each avatar type has a distinct presence — don't make them look the same.

| Type | Size | Ring style |
|---|---|---|
| Luna (owl) | 44px in chat, 72–80px hero | Animated conic-gradient (`#a78bfa → #4fc3f7 → #e879f9`) + float animation |
| Child profile | 64px | Warm gradient ring (`#4fc3f7 → #f59e0b → #a78bfa`) when active |
| Narrator (cast) | 72px | Gold gradient ring (`#f59e0b → #fbbf24`) + 👑 crown badge at top |
| Cast characters | 64px | Cyan-to-purple ring (`#4fc3f7 → #a78bfa`) |

`OwlAvatar` component is defined in `LunaChatPanel.tsx` and reused wherever Luna appears.

## Home Screen Layout (home/page.tsx)

Current section order (top → bottom):
1. Header (greeting + child switcher + journey heatmap strip)
2. **Hero banner** — Netflix-style featured Continue Listening story (full-width, 260px, cover art + progress bar + "▶ Continue" CTA)
3. **Tonight's Picks** — up to 2 user stories + 3 classics, mixed rail
4. **Continue Listening** — all in-progress stories with progress bars + duration badge
5. **Your Stories** — last 6 user stories
6. **Shared with Me** — empty state for now, shown lower because it's secondary
7. **Classics** — ready classic stories

**Design decisions made:**
- Duration badges removed from Tonight's Picks and Your Stories cards — too busy; only Continue Listening shows duration
- "Your Story" tag removed from Tonight's Picks cards — redundant noise; only "Classic" tag kept since it's genuinely informative when mixed with user stories
- Progress percentages in Continue Listening are **mock placeholders** (`[42, 68, 23]`) — real playback position tracking is not wired up yet
- Hero banner shows `continueStories[0]`; the rail below shows all stories (including the first one, so the user has a scrollable reference)
